## Краткое описание
Цель работы — построить воспроизводимый ML-пайплайн для раннего выявления нестабильности/аномалий в телеметрических данных воздушного движения (OpenSky Network) и сравнить несколько подходов к обнаружению аномалий в многомерных временных рядах.

Планируется использовать признаки из телеметрии (например: lat/lon, высота, скорость, курс/heading, вертикальная скорость), а также производные признаки (скорости изменения, ускорения, сглаженные тренды, статистики по окнам).  
Дополнительно (опционально) — учёт погодных факторов (например, через Meteostat).

**Выходы проекта:**
- воспроизводимый ML-пайплайн (сбор данных, подготовка, обучение, оценка)
- сравнительный отчёт по моделям и метрикам
- прототип мониторинга (дашборд) с отображением полётов/карты и risk-score/тревог

---

## Данные
Источник: OpenSky Network [API](https://opensky-network.org/). 

Первичная выборка: ограниченный период (примерно 1–2 недели) и один регион (для управляемого масштаба).  

Объекты анализа: траектории/сегменты полётов, представленные как многомерные временные ряды.

## План работ (верхнеуровневый)
### Этап 1 - Подготовка (Чекпойнт 1: репозиторий и план)
- Создание репозитория и структуры проекта
- Формулировка цели/задач, постановка экспериментов
- План, который будет уточняться по мере прогресса

### Этап 2 - Обзор источников и постановка задачи
- Обзор методов аномалий в временных рядах и в траекториях (ADS-B/aviation, time series anomaly detection)
- Выбор формализации: что считаем “аномалией/нестабильностью”, на каких уровнях (точка/окно/трека)
- Определение требований к метрикам и валидации (в условиях слабой разметки)

### Этап 3 - Сбор данных и подготовка датасета
- Получение данных через OpenSky API (ноутбук), сохранение таблиц для дальнейшей работы
- Очистка пропусков, выбросов сенсоров, несогласованности, ресемплинг/интерполяция (при необходимости)
- Feature engineering: производные признаки (скорости изменения, ускорения, статистики по окнам, признаки стабильности; здесь физически обоснованные фичи для самолётов, надо будет разбираться)
- Формирование обучающей/валидационной выборки (по времени/по траекториям)

### Этап 4 - EDA и базовая диагностика
- Анализ распределений, проверка на странности в данных
- Визуализация траекторий и ключевых признаков во времени
- Формулируем первичный список возможных "подозрительных" событий (гипотезы)

### Этап 5 - Baseline модели и метрики
- Простые baseline-подходы (правила/пороговые эвристики, простые статистические методы по окнам (возможно, метод трёх сигм))
- Выбор метрик для сравнения моделей (с учётом отсутствующей разметки, т.к. задача - обучение без учителя)
- Определение процедуры сравнения "чувствительность vs устойчивость" (robustness)

### Этап 6 - Продвинутые модели аномалий (сравнение)
- Isolation Forest (на агрегированных/оконных признаках)
- DBSCAN/HDBSCAN (кластеризация и детекция outlier в признаковом пространстве)
- LSTM Autoencoder (или иной sequence autoencoder) для многомерных рядов (посмотрим, что покажет себя лучше; может, в статьях что-то найду более интересное)
- Сравнение подходов по единому протоколу, анализ стабильности результатов

### Этап 7 - Risk-score и интерпретация
- Преобразование anomaly-score в калиброванный risk-score (калибровка/нормализация/пороговые уровни)
- Интерпретация вклада признаков, построение временные графиков, кейсов, которые удалось обнаружить
- (Опционально) погодные регрессоры/связь с погодой через Meteostat (если останется время и будет необходимость)

### Этап 8 - Прототип мониторинга (дашборд)
- Прототип в Streamlit/Dash: карта/траектории, графики признаков, risk-score, "alerts" (тревоги)
- Демонстрационный режим на выбранном регионе и периоде

### Этап 9 - Итоговый отчёт и финализация
- Сводные выводы по применимости методов к потоковым данным
- Ограничения, риски, рекомендации
- Финальная воспроизводимость: инструкции запуска, фиксированные зависимости, структура репо
